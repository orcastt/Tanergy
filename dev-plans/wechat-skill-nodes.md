# TANGENT — 公众号 Skill 节点设计规范

**版本**: v1.0  
**日期**: 2026-04-20  
**状态**: 已锁定，待开发

> 本文档专注于「公众号长文创作」Skill 所需的 7 个全新节点。
> 这些节点的设计打破了原有纯自动化 DAG 流水线模式，引入了「人工决策门」和「模板引擎」两个全新机制。

---

## 一、核心设计原则

### 1. 执行模型升级：人工决策门

原有节点系统是纯自动化 DAG——数据从左流到右，一气呵成。  
公众号 Skill 需要**两次强制人工暂停**，因此引入 `Gate` 节点。

**Gate 节点生命周期（Option C 已确认）**：

```
①  上游节点完成 → 数据传入 Gate
②  Gate 状态变为「等待中」（琥珀色边框脉冲）
③  画布上动态生成一个「临时交互节点」（动画出现，靠近 Gate）
④  用户在临时节点上操作（选择 / 输入）
⑤  临时节点淡出消失（200ms）
⑥  Gate 节点折叠显示结果：「✓ 已选：方向B」/ 「✓ 已输入（1,234字）」
⑦  Gate 进入「完成」状态（绿色），数据往下流，恢复执行
```

### 2. 模板引擎 vs AI 生成

`HTML Formatter` 节点是**确定性模板引擎**，不是 Chat 节点。  
Markdown → WeChat HTML 的映射关系是固定规则，不走 LLM。  
这保证每次输出的 HTML 结构稳定，品牌风格一致。

### 3. Gallery 多端口设计

`Image Gallery` 节点每张图都有独立输出端口，可以连接到  
`HTML Formatter` 的对应图片插槽，实现「哪张图放哪里」精确控制。

## 二、端口类型补充

在原有端口类型基础上，新增：

| 类型 | 颜色 | TS 格式 | 用途 |
|------|------|---------|------|
| `research_result` | 🟤 棕 `#92400E` | `ResearchResult` | 多轮搜索汇总（含信源） |
| `outline_options` | 🟣 紫（同 prompt） | `OutlineOption[]` | 选题方案列表，给 Gate 消费 |
| `image_slot` | 🟢 绿（同 image） | `ImageAsset \| null` | HTML 图片插槽，未连线为 null |

```typescript
interface ResearchResult {
  topic: string
  collected_at: string          // ISO 时间
  sources: { title: string; url: string; snippet: string }[]
  summary: string               // 汇总段落（Markdown）
  raw_results: SearchResult[]   // 所有原始搜索结果
}

interface OutlineOption {
  id: string
  title: string                 // 文章标题
  angle: string                 // 切入角度（1-2句）
  pros: string[]                // 优势
  cons: string[]                // 劣势
  sections: {
    heading: string
    word_count: number
  }[]                           // 粗略大纲
}
```

### 节点 1：`gate` — 人工决策门

**分期**: MVP（Skill 必须）· **计费**: 免费  
**是唯一一个会暂停画布执行的节点**

#### 端口

| 端口 ID | 方向 | 类型 | 说明 |
|--------|------|------|------|
| `options_in` | 输入 | `outline_options` | 选题方案（select 模式） |
| `prompt_in` | 输入 | `text` | 告诉用户需要提供什么（input 模式） |
| `out` | 输出 | `text` | 用户选择/输入的内容（序列化为文本） |

#### 节点配置项

- `Mode`：`select`（选择门）/ `input`（输入门）
- `Title`：临时节点的标题文字（如「请选择选题方向」）

#### 两种模式的临时节点 UI

**select 模式**（由 Outline Generator 驱动）：
```
┌──────────────────────────────────────┐
│  📋 请选择选题方向                    │
│  ────────────────────────────────── │
│  ┌──────────────────────────────┐   │
│  │ A  [标题A]                   │   │
│  │    [角度描述一句话]     [选择]│   │
│  ├──────────────────────────────┤   │
│  │ B  [标题B]                   │   │
│  │    [角度描述一句话]     [选择]│   │
│  ├──────────────────────────────┤   │
│  │ C  [标题C]                   │   │
│  │    [角度描述一句话]     [选择]│   │
│  └──────────────────────────────┘   │
└──────────────────────────────────────┘
```

**input 模式**（等用户提交素材）：
```
┌──────────────────────────────────────┐
│  ✍️ 提供你的真实素材                  │
│  ────────────────────────────────── │
│  系统需要：                           │
│  · 你的个人使用体验（具体细节）        │
│  · 真实数据/截图描述                   │
│  · 你的核心观点                       │
│  ────────────────────────────────── │
│  ┌──────────────────────────────┐   │
│  │ 在这里粘贴你的素材...        │   │
│  │                              │   │
│  │                              │   │
│  └──────────────────────────────┘   │
│                        [确认提交 →]  │
└──────────────────────────────────────┘
```

#### 节点状态显示

| 状态 | 边框 | 节点内容 |
|------|------|---------|
| `idle` | 默认 | 「等待上游数据...」 |
| `waiting` | 琥珀色脉冲 `#F59E0B` | 「⏸ 等待你的选择」 |
| `done` | 绿色 | `✓ 已选：[标题A]` 或 `✓ 已输入（1,234字）` |

### 节点 2：`research` — 深度调研

**分期**: MVP（Skill 必须）· **计费**: 按调用次数（5次搜索 = 5次计费）

#### 端口

| 端口 ID | 方向 | 类型 | 说明 |
|--------|------|------|------|
| `query` | 输入 | `text` 或 `prompt` | 调研主题关键词 |
| `out` | 输出 | `research_result` | 汇总调研报告（含信源） |

#### 节点配置项

- `搜索轮次`：3 / 5 / 8（默认 5）
- `信源语言`：中文优先 / 英文优先 / 混合
- `时间范围`：最近一周 / 一月 / 三月 / 不限

#### 执行行为

1. 根据输入主题，自动拆解为 N 个不同角度的搜索 query
2. 并发执行 N 次 Tavily 搜索
3. 用 Claude 汇总所有结果，去重、提炼，标注信源
4. 输出结构化 `ResearchResult`

#### 节点内显示（完成后）

```
┌─────────────────────────────────┐
│ 🔍 Research             ✓ 完成  │
│ 共搜索 5 次 · 12 个信源         │
│ ─────────────────────────────  │
│ 调研摘要（前3行）...            │
│                     [查看全文 ↗] │
└─────────────────────────────────┘
```

### 节点 3：`outline_generator` — 选题生成器

**分期**: MVP · **计费**: 按 Token

#### 端口

| 端口 ID | 方向 | 类型 | 说明 |
|--------|------|------|------|
| `brief` | 输入 | `text` | 用户需求简介 |
| `research` | 输入 | `research_result` | 调研报告 |
| `out` | 输出 | `outline_options` | 3-4 个选题方案 |

#### 执行行为

调用 Claude，生成 3-4 个 `OutlineOption`，包含：
- 吸引人的标题
- 一句话切入角度
- 优劣势分析
- 章节大纲（标题 + 预计字数）

#### 节点内显示（完成后）

```
┌─────────────────────────────────┐
│ 📑 Outline Generator    ✓ 完成  │
│ 生成 3 个选题方向               │
│ ─────────────────────────────  │
│ A. [标题A]                      │
│ B. [标题B]                      │
│ C. [标题C]                      │
└─────────────────────────────────┘
```

下游 Gate（select 模式）消费此输出，展示给用户选择。

### 节点 4：`writer` — 长文写作引擎

**分期**: MVP · **计费**: 按 Token

#### 端口

| 端口 ID | 方向 | 类型 | 说明 |
|--------|------|------|------|
| `outline` | 输入 | `text` | 选定的大纲（来自 Gate 输出） |
| `research` | 输入 | `research_result` | 调研报告 |
| `materials` | 输入 | `text` | 用户真实素材（来自 Gate 输入） |
| `out` | 输出 | `text` | Markdown 初稿 |

#### 节点配置项

- `目标字数`：1000 / 2000 / 3000 / 5000（默认 3000）
- `写作平台`：公众号 / 小红书 / 通用长文
- `写作风格`：深度解析 / 轻松科普 / 情感共鸣 / 干货清单

#### 内置写作规则（不暴露给用户，System Prompt 内置）

```
- 用真实经历引入，不用套话开头
- 所有数据和案例必须来自 research 或 materials，不得编造
- 禁止使用：在当今时代、综上所述、值得注意的是、不缺...缺的是...
- 超过30字的长句必须拆短
- 每段不超过5行（手机屏幕友好）
- 口语化，避免书面词汇
```

### 节点 5：`reviewer` — 三遍审校链

**分期**: MVP · **计费**: 按 Token（内部 3 次 Claude 调用）

#### 端口

| 端口 ID | 方向 | 类型 | 说明 |
|--------|------|------|------|
| `draft` | 输入 | `text` | 初稿 Markdown |
| `research` | 输入 | `research_result` | 调研报告（用于事实核查） |
| `out` | 输出 | `text` | 三遍审校后的成稿 Markdown |

#### 执行行为（内部，用户不可见）

```
Pass 1（事实核查）
  Claude: 核对所有数据、时间、产品名称
  → 输出修正版文本

Pass 2（反AI洗稿）  
  Claude: 删除套话、拆解排比、口语化、加入个人态度
  风格参考：Keso/和菜头/梁宁的句式节奏，只借风格不引原话
  → 输出去AI化文本

Pass 3（节奏格式）
  Claude: 拆长句（>30字）、控制段落长度（≤5行）、标点优化
  → 输出最终成稿
```

#### 节点内显示

```
执行中：

│ ✏️ Reviewer               运行中 │
│ ██████████░░░░░░░░░░  Pass 2/3  │
│ 正在：反AI洗稿...               │
└─────────────────────────────────┘

完成后：
┌─────────────────────────────────┐
│ ✏️ Reviewer               ✓完成  │
│ 3遍审校完成 · 共修改 47处       │
│ ─────────────────────────────  │
│ 最终成稿前3行...                │
```

## 五、开发顺序建议

这 7 个节点有依赖关系，建议按以下顺序开发：

| 顺序 | 节点 | 原因 |
|------|------|------|
| 1 | `gate` | 最核心的架构创新，其他节点依赖它的「暂停/恢复」机制 |
| 2 | `research` | 相对独立，只依赖 Tavily + Claude |
| 3 | `outline_generator` | 依赖 research 输出格式 |
| 4 | `writer` | 依赖 gate 输出、research、用户素材 |
| 5 | `reviewer` | 依赖 writer 输出 |
| 6 | `image_gallery` | 多端口机制，需要画布连线支持 |
| 7 | `html_formatter` | 最复杂：模板引擎 + 图片插槽 + 品牌编辑器 |

**前置条件（Slice 3 画布核心必须先完成）**：
- 节点状态机（idle/running/done/error/waiting）
- 画布连线（端口拖拽、类型校验）
- WebSocket 执行推送
- **新增状态 `waiting`（Gate 专用）**：执行暂停、等待用户交互、恢复执行
